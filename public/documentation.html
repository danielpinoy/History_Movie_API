List of API Endpoints

  • Endpoint 1: Get All Movies

    Description: Returns a list of all historic movies.

    Endpoint URL: /historic-movies

    Method Type: GET

    Query Parameters: None

    Request Body: N/A

    Response Data: JSON array representing a list of movies.
    Example: 

    [
    {
        "title": "Babylon",
        "director": "Damien Chazelle",
        "criticRating": 9.0,
        "userRating": 8.5,
        "genres": ["Drama", "Musical"]
    },
    // ... other movies ...
]

  • Endpoint 2: Get Movie by Title

    Description: Returns a single movie based on its title.

    Endpoint URL: /historic-movies/:title

    Method Type: GET

    Query Parameters: None

    Request Body: N/A

    Response Data: JSON object representing the movie.
    Example: 

    {
    "title": "Babylon",
    "director": "Damien Chazelle",
    "criticRating": 9.0,
    "userRating": 8.5,
    "genres": ["Drama", "Musical"]
}

  • Endpoint 3: Get Movies by Genre

    Description: Returns a list of movies based on a specified genre.

    Endpoint URL: /historic-movies/genre/:genres

    Method Type: GET

    Query Parameters: None

    Request Body: N/A

    Response Data: JSON array representing a list of movies.
    Example: 

    [
	{
			"title": "Babylon",
			"director": "Damien Chazelle",
			"criticRating": 9.0,
			"userRating": 8.5,
			"genres": ["Drama", "Musical"]
	},
	// ... other movies ...
]

  • Endpoint 4: Get Director by Name

    Description: Returns information about a movie director based on their name.

    Endpoint URL: /historic-movies/director/:name

    Method Type: GET

    Query Parameters: None

    Request Body: N/A

    Response Data: JSON object representing the director's information.
    Example: 

    {
	"name": "Damien Chazelle",
	"bio": "Damien Chazelle is an American director known for...",
	"birthYear": 1985
}

  • Endpoint 5: Get All Users

    Description: Retrieves a list of all registered users.

    Endpoint URL: /users

    Method Type: GET

    Query Parameters: None

    Request Body: N/A

    Response Data: JSON array containing user objects, each with an id, username, and password.
    Example: 

    [
    {
        "id": 1,
        "username": "user1",
        "password": "password123"
    },
    {
        "id": 2,
        "username": "user2",
        "password": "password456"
    },
    
]

  • Endpoint 6: Add Movie to User's Favorites

    Description: Allows a user to add a movie to their list of favorite movies.

    Endpoint URL: /users/:id/:movieTitle

    Method Type: POST

    Query Parameters: None

    Request Body: N/A

    Response Data: Text confirmation that the movie has been added.
    Example: 

    "Gladiator has been added to user 1's favorites"

  • Endpoint 7: Remove Movie from User's Favorites

    Description: Allows a user to remove a movie from their list of favorite movies.

    Endpoint URL: /users/:id/:movieTitle

    Method Type: DELETE

    Query Parameters: None

    Request Body: N/A

    Response Data: Text confirmation that the movie has been removed.
    Example: 

    "Gladiator has been removed from user 1's favorites"

  • Endpoint 8: Delete User

    Description: Deletes a user's account and associated data.

    Endpoint URL: /users/:id/

    Method Type: DELETE

    Query Parameters: None

    Request Body: N/A

    Response Data: Text confirmation of the user's deletion.
    Example: 

    "User 1 has been removed"

  • Endpoint 9: Get Documentation

    Description: Retrieves the API documentation HTML page.

    Endpoint URL: /documentation

    Method Type: GET

    Query Parameters: None

    Request Body: N/A

    Response Data: HTML content of the API documentation page.

  • Endpoint 10: Add New User

    Description: Registers a new user.

    Endpoint URL: /register

    Method Type: POST

    Query Parameters: None

    Request Body: JSON object with the following fields:

    • username: The username of the new user (string).
    • password: The password of the new user (string).
    Example: 
    {
	"username": "new_user",
	"password": "password123"
}

    Response Data: JSON object with a success message.
    Example: 

    {
	"message": "User registered successfully"
}

  • Endpoint 11: Update User Username

    Description: Updates a user's username.

    Endpoint URL: /users/:id

    Method Type: PUT

    Request Parameters:

    • :id (string) - The unique identifier of the user to update.

    Request Body: 

    {
	"newUsername": "new_username_here"
}

    Response Data: JSON object containing a message and the updated user object.
    Example: 

    {
	"message": "Username updated successfully",
	"userUpdate": {
			"id": "user_id_here",
			"username": "new_username_here",
			"password": "user_password_here"
	}
}
