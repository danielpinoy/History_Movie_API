List of API Endpoints

List of API Endpoints

  • Endpoint 1: Get All Movies

    Description: Returns a list of all historic movies.

    Endpoint URL: /Movies

    Method Type: GET

    Query Parameters: None

    Request Body: N/A

    Response Data: JSON array representing a list of movies.

    Sample Request:
    To retrieve all historic movies, make a GET request to: http://your-api-url/Movies

    Response Status Codes:
    - 200 (OK): Successful request, movies data is returned.
    - 404 (Not Found): No historic movies found.

  • Endpoint 2: Register a User

    Description: Register a new user.

    Endpoint URL: /register

    Method Type: POST

    Request Body: JSON object with user data. Example: 

    {
	"Username": "newuser",
	"Password": "password123",
	"Email": "newuser@example.com",
	"Birthday": "1990-01-15",
	"FavoriteMovies": []
}

    Response Data: JSON object representing the newly registered user. Example: 

    {
	"_id": "603b1304f88c9d5dfe837b33",
	"Username": "newuser",
	"Email": "newuser@example.com",
	"Birthday": "1990-01-15",
	"FavoriteMovies": []
}

    Response Status Codes:
    - 201 (Created): User registration successful.
    - 400 (Bad Request): User with the same username already exists.
    - 500 (Internal Server Error): Registration failed.

  • Endpoint 3: Update User Info

    Description: Update user information.

    Endpoint URL: /Users/:Username

    Method Type: PUT

    Request Body: JSON object with user data to be updated. Example: 

    {
"Username": "newusername",
"Password": "newpassword",
"Email": "newemail@example.com",
"Birthday": "1995-02-20"
}

    Response Data: JSON object representing the updated user. Example: 

    {
"_id": "603b1304f88c9d5dfe837b33",
"Username": "newusername",
"Email": "newemail@example.com",
"Birthday": "1995-02-20",
"FavoriteMovies": []
}

    Response Status Codes:
    - 200 (OK): User information updated successfully.
    - 404 (Not Found): User not found.
    - 500 (Internal Server Error): Update failed.

  • Endpoint 4: Add Favorite Movie to User

    Description: Add a movie to a user's list of favorite movies.

    Endpoint URL: /Users/addfavorite/

    Method Type: POST

    Request Body: 

    						{
						"userId": "652ba985332f7cee65d16733",
						"movieId": "652b9a3d332f7cee65d16731"
				}

    Response Data: JSON object representing the updated user with the added movie. Example: 

    {
"_id": "603b1304f88c9d5dfe837b33",
"Username": "newusername",
"Email": "newemail@example.com",
"Birthday": "1995-02-20",
"FavoriteMovies": ["603b1304f88c9d5dfe837b34"]
}

    Response Status Codes:
    - 200 (OK): Movie added to favorites successfully.
    - 404 (Not Found): User not found.
    - 500 (Internal Server Error): Adding movie to favorites failed.

  • Endpoint 5: Remove Favorite Movie from User

    Description: Remove a movie from a user's list of favorite movies.

    Endpoint URL: /Users/:id/:movieTitle

    Method Type: DELETE

    Request Body: N/A

    Response Data: JSON object indicating whether the movie was removed from favorites. Example: 

    {
"message": "Movie removed from favorites."
}

    Response Status Codes:
    - 200 (OK): Movie removed from favorites successfully.
    - 404 (Not Found): User not found or movie not in favorites.
    - 500 (Internal Server Error): Removing movie from favorites failed.

  • Endpoint 6: Delete User

    Description: Delete a user account.

    Endpoint URL: /Users/:id

    Method Type: DELETE

    Request Body: N/A

    Response Data: JSON object indicating whether the user was deleted. Example: 

    {
"message": "User deleted successfully"
}

    Response Status Codes:
    - 200 (OK): User deleted successfully.
    - 404 (Not Found): User not found.
    - 500 (Internal Server Error): User deletion failed.

  • Endpoint 7: Get All Movies

    Description: Get a list of all historic movies.

    Endpoint URL: /Movies

    Method Type: GET

    Request Body: N/A

    Response Data: JSON array representing a list of movies. Example: 

    [
{
	"title": "Babylon",
	"director": "Damien Chazelle",
	"criticRating": 9.0,
	"userRating": 8.5,
	"genres": ["Drama", "Musical"]
},
// ... other movies ...
]

    Response Status Codes:
    - 200 (OK): Successful request, movies data is returned.
    - 404 (Not Found): No historic movies found.

  • Endpoint 8: Get Movie by Title

    Description: Get a historic movie by its title.

    Endpoint URL: /Movies/:Title

    Method Type: GET

    Request Body: N/A

    Response Data: JSON object representing the movie. Example: 

    {
"title": "Babylon",
"director": "Damien Chazelle",
"criticRating": 9.0,
"userRating": 8.5,
"genres": ["Drama", "Musical"]
}

    Response Status Codes:
    - 200 (OK): Movie data is returned successfully.
    - 404 (Not Found): Movie with the given title not found.

  • Endpoint 9: Get Movies by Genre

    Description: Get historic movies by genre.

    Endpoint URL: /Movies/genres/:Genre

    Method Type: GET

    Request Body: N/A

    Response Data: JSON array representing a list of movies of the specified genre. Example: 

    [
{
	"title": "Babylon",
	"director": "Damien Chazelle",
	"criticRating": 9.0,
	"userRating": 8.5,
	"genres": ["Drama", "Musical"]
},
// ... other movies of the specified genre ...
]

    Response Status Codes:
    - 200 (OK): Movies data is returned successfully.
    - 404 (Not Found): Movies with the given genre not found.

  • Endpoint 10: Get Movies by Director

    Description: Get historic movies by director.

    Endpoint URL: /Movies/Director/:Name

    Method Type: GET

    Request Body: N/A

    Response Data: JSON object representing the director's filmography. Example: 

    {
"Name": "Damien Chazelle",
"Movies": [
	"Babylon",
	"La La Land",
]
}

    Response Status Codes:
    - 200 (OK): Director's filmography is returned successfully.
    - 404 (Not Found): Director with the given name not found.
